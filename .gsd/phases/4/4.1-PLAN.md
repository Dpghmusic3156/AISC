# Plan 4.1: Result Window UI & Markdown Support

## Objective
Replace the raw `MessageBox` with a custom, borderless, floating WPF Window that elegantly renders the AI's Markdown response, allowing the user to copy text or close the result comfortably.

## Context
- .gsd/SPEC.md
- src/AIScreenCapture.UI/App.xaml.cs (Current MessageBox implementation)

## Tasks

<task type="auto">
  <name>Install Markdown Library</name>
  <files>src/AIScreenCapture.UI/AIScreenCapture.UI.csproj</files>
  <action>
    Add the `MdXaml` NuGet package to the UI project to support rendering Markdown text directly inside WPF control surfaces.
    - Run `dotnet add src/AIScreenCapture.UI package MdXaml`
  </action>
  <verify>dotnet build src/AIScreenCapture.UI</verify>
  <done>MdXaml Package Reference exists in the csproj file.</done>
</task>

<task type="auto">
  <name>Create ResultWindow.xaml</name>
  <files>
    src/AIScreenCapture.UI/Views/ResultWindow.xaml
    src/AIScreenCapture.UI/Views/ResultWindow.xaml.cs
  </files>
  <action>
    Create a new WPF Window `ResultWindow` in the Views folder.
    - `WindowStyle="None"`, `AllowsTransparency="True"`, `Topmost="True"`
    - Use a MaterialDesign Card with rounded corners and a slight drop shadow.
    - Add a header with a "Drag" thumb/area, a title (e.g., "AI Result"), and a "Close" (X) button.
    - Add a `ScrollViewer` containing the `MarkdownScrollViewer` (from MdXaml) to render the response.
    - Add a "Copy to Clipboard" button at the bottom right.
    - Implement `MouseLeftButtonDown` on the header to allow `DragMove()`.
    - Provide a public method `SetResult(string markdown, string modelName)` to bind data.
  </action>
  <verify>dotnet build src/AIScreenCapture.UI</verify>
  <done>ResultWindow is created, builds without errors, and has a method to inject markdown text.</done>
</task>

## Success Criteria
- [ ] MdXaml is successfully installed.
- [ ] Built a custom floating WPF window.
- [ ] The window supports dragging, closing, copying to clipboard, and Markdown rendering.
