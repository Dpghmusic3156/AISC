# Plan 1.4: Dummy Popup UI

## Objective
Set up the floating Popup window template that will eventually show AI results, ensuring it behaves correctly as an overlay (draggable, borderless).

## Context
- This sets the stage for Phase 4 where data will actually be parsed. Right now, it just needs to look right.

## Tasks

<task type="auto">
  <name>Create Popup Window Structure</name>
  <files>
    - src/AIScreenCapture.UI/Views/ResultPopupWindow.xaml
    - src/AIScreenCapture.UI/Views/ResultPopupWindow.xaml.cs
  </files>
  <action>
    - Create `ResultPopupWindow` inherited from `Window`.
    - Set `WindowStyle="None"`, `AllowsTransparency="True"`, `Background="Transparent"`, `Topmost="True"`.
    - In the XAML, create a Main Border with rounded corners (e.g. `CornerRadius="8"`), a shadow effect (`DropShadowEffect`), and a solid dark/light Material Design background.
    - Add a header area (like a subtle grip handle) that handles window dragging (`MouseLeftButtonDown` -> `DragMove()`).
    - Add a dummy TextBlock for the AI response and a MaterialDesign `ProgressBar` (`IsIndeterminate="True"`) to simulate loading.
    - Add a close button (X) that hides or closes the prompt window.
  </action>
  <verify>dotnet build src/AIScreenCapture.UI/AIScreenCapture.UI.csproj</verify>
  <done>ResultPopupWindow XAML is successfully built with appropriate transparency and drag behavior.</done>
</task>

## Success Criteria
- [ ] ResultPopupWindow can be spawned via tray menu.
- [ ] Window is borderless, has shadow and rounded corners, and sits on top of other windows.
- [ ] User can click and drag the window around the screen.
- [ ] Closing the popup hides it without crashing the background app.
