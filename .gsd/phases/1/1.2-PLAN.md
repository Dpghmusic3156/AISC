# Plan 1.2: Settings Schema & Storage

## Objective
Implement the configuration logic to save and load API keys and custom presets to/from a `config.json` file in the user's local application data folder.

## Context
- .gsd/SPEC.md
- Use standard `System.Text.Json` for serialization.

## Tasks

<task type="auto">
  <name>Create Configuration Models</name>
  <files>
    - src/AIScreenCapture.Core/Models/AIProvider.cs
    - src/AIScreenCapture.Core/Models/AIPreset.cs
    - src/AIScreenCapture.Core/Models/AppSettings.cs
  </files>
  <action>
    - Define `AIProvider` as an enum (OpenAI, Gemini, Claude).
    - Define `AIPreset` with Name (string), Provider (AIProvider), ModelName (string), and SystemPrompt (string).
    - Define `AppSettings` with fields for ApiKeyOpenAI, ApiKeyGemini, ApiKeyClaude (all strings), and a `List<AIPreset>` Presets.
    - Set the default constructor of `AppSettings` to include the default preset: "Giải thích nội dung trong ảnh này", Provider = Gemini, ModelName = "gemini-3-flash".
  </action>
  <verify>dotnet build src/AIScreenCapture.Core/AIScreenCapture.Core.csproj</verify>
  <done>Models exist and compile successfully.</done>
</task>

<task type="auto">
  <name>Implement Settings Manager</name>
