# Plan 1.3: Configuration Window UI

## Objective
Build the main configuration window using MaterialDesignInXAML where users can edit their API keys and manage custom presets.

## Context
- Requires `1.2-PLAN.md` settings manager to be complete.
- MVVM pattern should be followed loosely or strictly depending on preference, but DataBinding is required.

## Tasks

<task type="auto">
  <name>Build Configuration Window XAML</name>
  <files>
    - src/AIScreenCapture.UI/Views/ConfigWindow.xaml
    - src/AIScreenCapture.UI/Views/ConfigWindow.xaml.cs
    - src/AIScreenCapture.UI/ViewModels/ConfigViewModel.cs
  </files>
  <action>
    - Create a standard WPF Window `ConfigWindow`.
    - Implement a View Model (`ConfigViewModel`) that loads `AppSettings` via `SettingsManager`.
    - Create a tabbed interface or scrollable dialog with two main sections:
      1. API Keys: Password boxes or textboxes for OpenAI, Gemini, Claude.
      2. Presets: A ListBox/DataGrid showing `Presets`, and basic fields (Name, Provider Dropdown, Model, Prompt) to Add/Remove items.
    - Add a "Save" button that triggers `SettingsManager.Save()`.
    - Apply `MaterialDesignThemes` styles to inputs, buttons, and the window background.
  </action>
  <verify>dotnet build src/AIScreenCapture.UI/AIScreenCapture.UI.csproj</verify>
  <done>Config window XAML is written and builds.</done>
</task>

- [ ] Configuration window can be opened via the tray icon.
- [ ] Values typed into the API key fields are correctly bound and saved to `config.json` upon saving.
- [ ] The default preset is visible in the Presets list.
